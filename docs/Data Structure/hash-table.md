- 해시 테이블은 **키-값 쌍**을 저장하는 자료구조
- 해시 함수를 사용해 키를 배열의 인덱스로 변환하여 데이터를 저장

## 시간 복잡도

- 평균: O(1) - 삽입, 삭제, 검색 모두
- 최악: O(n) - 충돌이 많을 때

## 해시 함수

- 키를 입력받아 배열 인덱스(해시값)를 반환하는 함수
- 좋은 해시 함수는 데이터를 균등하게 분산시켜 충돌을 최소화

## 충돌 해결 방법

1. 체이닝 (Chaining)
   같은 인덱스에 여러 값이 들어오면 연결 리스트로 연결해서 저장

   ```js
   // 같은 인덱스에 연결 리스트로 여러 값 저장
   hashTable[3] = [
     ["apple", "사과"],
     ["grape", "포도"], // 같은 인덱스에 체인으로 연결
   ];
   ```

2. 개방 주소법 (Open Addressing)

   - 충돌 발생 시 다른 빈 공간을 찾아 저장
   - 선형 탐사, 이차 탐사, 이중 해싱 등의 방법이 있음

   ```js
   // 충돌 시 다음 빈 공간을 찾아서 저장
   function set(key, value) {
     let index = hash(key) % size;

     while (hashTable[index] !== null) {
       index = (index + 1) % size; // 다음 인덱스로 이동
     }

     hashTable[index] = [key, value];
   }
   ```

## 활용

중복 제거: Set 자료 구조의 구현

- Python의 딕셔너리 (dict)
- JavaScript의 객체 (Object), Map
- Java의 HashMap
- 데이터베이스 인덱싱
- 캐시 구현
- 패스워드 저장 (암호화된 해시값으로 저장)
- 체크섬 (데이터 무결성 검증)

## 장단점

장점: 빠른 검색/삽입/삭제 속도  
단점: 해시 충돌 가능성, 순서 보장 안 됨, 메모리 오버헤드
