## CDN이란?

콘텐츠를 전 세계 여러 서버에 분산 저장하고, 사용자와 가장 가까운 서버에서 콘텐츠를 제공하는 분산 서버 네트워크 시스템

- 트래픽이 많지 않고 국내 방문자만 있는 웹사이트는 CDN을 도입할 필요가 없음
- 웹사이트 지연 로딩이 높거나 트래픽이 많고 서버 성능 이슈가 발견되면 CDN 도입 고려

## 동작 원리

1. 사용자가 콘텐츠 요청
2. DNS가 가장 가까운 엣지 서버 반환
   - DNS 서버: CDN에서 오리진 서버와 엣지 서버를 연결해주는 역할이며, 사용자가 오리진 서버에 요청을 보내면 DNS 서버는 사용자를 가장 가까운 엣지 서버로 라우팅해줌
3. 엣지 서버에서 콘텐츠 제공
   - 엣지 서버: 전 세계 곳곳 위치한 분산 서버이며 오리진 서버의 콘텐츠를 캐싱하고 있음 (CDN 공급업체가 관리)
4. 캐시 없으면 오리진 서버에서 가져옴
   - 오리진 서버: 웹 사이트의 원본 콘텐츠를 저장하고 있는 중앙 서버이며, 웹사이트 운영자는 오리진 서버에서 콘텐츠를 업데이트하거나 삭제하고, 오리진 서버를 관리해야 함

## 장점

- 빠른 콘텐츠 전송 (낮은 지연시간)
- 서버 부하 분산
- DDoS 공격 방어

## 주요 CDN 서비스

- Cloudflare
- AWS CloudFront
- Vercel Edge Network
