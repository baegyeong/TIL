## HTTP(HyperText Transfer Protocol)란?

- 웹에서 클라이언트와 서버 간에 데이터를 주고받기 위한 애플리케이션 계층 프로토콜
- HTML 문서, 이미지, 영상, JSON 등 다양한 리소스를 전송할 수 있음
- **비연결성(Connectionless)**: 요청에 대한 응답을 보낸 후 연결을 끊음
- **무상태성(Stateless)**: 각 요청은 독립적이며 이전 요청의 상태를 기억하지 않음

## HTTP 요청 구조

```
GET /index.html HTTP/1.1
Host: www.example.com
User-Agent: Mozilla/5.0
Accept: text/html
```

| 구성 요소            | 설명                                           |
| -------------------- | ---------------------------------------------- |
| 시작 줄 (Start Line) | HTTP 메서드, 요청 URL(타겟), HTTP 버전         |
| 헤더 (Headers)       | 요청에 대한 메타데이터 (Host, Content-Type 등) |
| 빈 줄                | 헤더와 본문을 구분                             |
| 본문 (Body)          | 전송할 데이터 (POST, PUT 등에서 사용)          |

## HTTP 응답 구조

```
HTTP/1.1 200 OK
Content-Type: text/html; charset=UTF-8
Content-Length: 1234

<html>...</html>
```

| 구성 요소             | 설명                                                 |
| --------------------- | ---------------------------------------------------- |
| 상태 줄 (Status Line) | HTTP 버전, 상태 코드, 상태 메시지                    |
| 헤더 (Headers)        | 응답에 대한 메타데이터 (Content-Type, Set-Cookie 등) |
| 빈 줄                 | 헤더와 본문을 구분                                   |
| 본문 (Body)           | 실제 응답 데이터                                     |

## HTTP 메서드

| 메서드  | 용도                                 | 멱등성 | 안전성 |
| ------- | ------------------------------------ | ------ | ------ |
| GET     | 리소스 조회                          | O      | O      |
| POST    | 리소스 생성                          | X      | X      |
| PUT     | 리소스 전체 수정                     | O      | X      |
| PATCH   | 리소스 부분 수정                     | X      | X      |
| DELETE  | 리소스 삭제                          | O      | X      |
| HEAD    | GET과 동일하나 본문 없이 헤더만 반환 | O      | O      |
| OPTIONS | 서버가 지원하는 메서드 목록 조회     | O      | O      |

- **멱등성(Idempotent)**: 같은 요청을 여러 번 해도 결과가 동일
- **안전성(Safe)**: 서버의 상태를 변경하지 않음

## HTTP 상태 코드

### 2xx - 성공

| 코드           | 의미                               |
| -------------- | ---------------------------------- |
| 200 OK         | 요청 성공                          |
| 201 Created    | 리소스 생성 성공 (POST)            |
| 204 No Content | 성공했지만 응답 본문 없음 (DELETE) |

### 3xx - 리다이렉션

| 코드                  | 의미                                          |
| --------------------- | --------------------------------------------- |
| 301 Moved Permanently | 리소스가 영구적으로 이동됨                    |
| 302 Found             | 리소스가 임시로 이동됨                        |
| 304 Not Modified      | 캐시된 리소스가 유효함, 본문 없이 헤더만 반환 |

### 4xx - 클라이언트 오류

| 코드             | 의미                    |
| ---------------- | ----------------------- |
| 400 Bad Request  | 잘못된 요청 문법        |
| 401 Unauthorized | 인증 필요               |
| 403 Forbidden    | 인증은 됐지만 권한 없음 |
| 404 Not Found    | 리소스를 찾을 수 없음   |
| 409 Conflict     | 현재 서버 상태와 충돌   |

### 5xx - 서버 오류

| 코드                      | 의미                            |
| ------------------------- | ------------------------------- |
| 500 Internal Server Error | 서버 내부 오류                  |
| 502 Bad Gateway           | 게이트웨이/프록시 오류          |
| 503 Service Unavailable   | 서버 일시적 과부하 또는 점검 중 |

## 주요 HTTP 헤더

### 요청 헤더

| 헤더            | 설명                                       |
| --------------- | ------------------------------------------ |
| `Host`          | 요청하는 서버의 도메인 (HTTP/1.1에서 필수) |
| `Authorization` | 인증 정보 (Bearer 토큰 등)                 |
| `Content-Type`  | 요청 본문의 미디어 타입                    |
| `Accept`        | 클라이언트가 처리 가능한 미디어 타입       |
| `Cookie`        | 서버에서 설정한 쿠키 전송                  |

### 응답 헤더

| 헤더                          | 설명                                 |
| ----------------------------- | ------------------------------------ |
| `Content-Type`                | 응답 본문의 미디어 타입              |
| `Set-Cookie`                  | 클라이언트에 쿠키 설정               |
| `Cache-Control`               | 캐시 동작 방식 지정                  |
| `Location`                    | 리다이렉트할 URL (3xx 응답에서 사용) |
| `Access-Control-Allow-Origin` | CORS 허용 출처                       |

## 비연결성과 무상태성 보완

HTTP는 비연결성·무상태성으로 인해 서버 자원을 효율적으로 사용하지만, 매번 인증 정보를 다시 보내야 하는 문제가 있음

이를 보완하는 방법:

- **쿠키(Cookie)**: 클라이언트에 상태 저장, 요청마다 자동으로 서버에 전송
- **세션(Session)**: 서버에 상태 저장, 클라이언트는 세션 ID만 보유
- **토큰(Token)**: JWT 등을 사용해 무상태로 인증 정보를 전달
