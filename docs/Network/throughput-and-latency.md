## 처리량

단위 시간당 네트워크를 통해 실제로 전송되는 데이터의 양

- 단위: bps, Mbps, Gbps 등
- 대역폭과의 비교
  - 대역폭: 이론적 최대 전송 용량
  - 처리량: 실제로 흐른 데이터 양

### 처리량에 영향을 미치는 요소

| 요소                      | 설명                                     |
| ------------------------- | ---------------------------------------- |
| 네트워크 혼잡(Congestion) | 트래픽이 몰리면 처리량 감소              |
| 패킷 손실                 | 재전송이 발생해 실질 처리량 저하         |
| 프로토콜 오버헤드         | TCP 헤더, ACK 등이 실제 데이터 비율 감소 |
| 하드웨어 한계             | NIC, 라우터, 스위치의 물리적 한계        |

## 지연시간

데이터가 출발지에서 목적지까지 이동하는 데 걸리는 시간

- 단위: ms (밀리초)
- ping 값으로 확인 가능
- RTT(Round-Trip Time): 요청을 보내고 응답을 받을 때까지의 총 왕복 시간

### 지연 시간의 구성 요소

```
총 지연 = 전파 지연 + 전송 지연 + 처리 지연 + 큐잉 지연
```

| 종류                           | 설명                                    | 예시                          |
| ------------------------------ | --------------------------------------- | ----------------------------- |
| 전파 지연 (Propagation Delay)  | 신호가 물리적 매체를 통해 이동하는 시간 | 광섬유의 빛의 속도 한계       |
| 전송 지연 (Transmission Delay) | 패킷을 링크에 밀어 넣는 시간            | 패킷 크키 / 대역폭            |
| 처리 지연 (Processing Delay)   | 라우터/스위치가 패킷을 처리하는 시간    | 헤더 분석, 라우팅 테이블 조회 |
| 큐잉 지연 (Queuing Delay)      | 버퍼에서 대기하는 시간                  | 네트워크 혼잡 시 급증         |

- output link: 라우터가 패킷을 다음 목적지로 보내기 위해 사용하는 물리적 네트워크 회선
- 라우터: 둘 이상의 서로 다른 네트워크를 연결하고, 데이터 패킷을 목적지까지 가장 빠르고 효율적인 경로로 안내(라우팅)하는 장비
- 버퍼: 데이터를 한 곳에서 다른 곳으로 전송하는 동안 일시적으로 데이터를 보관하는 영역
- 패킷: 컴퓨터 네트워크를 통해 전송되는 데이터를 작게 나눈 조각

## 처리량 vs 지연 시간 비교

| 구분           | 처리량              | 지연 시간            |
| -------------- | ------------------- | -------------------- |
| 측정 대상      | 데이터 양           | 데이터 전달 시간     |
| 단위           | bps, Mbps, Gbps     | ms                   |
| 높을수록       | 좋음                | 나쁨                 |
| 주요 영향 요소 | 대역폭, 혼잡, 손실  | 거리, 흡수, 혼잡     |
| 중요한 서비스  | 파일 전송, 스트리밍 | 게임, VoIP, 금융거래 |
