title: 메모리 구조
---

프로그램 실행 시 실행파일을 메모리에 적재한다.

- 텍스트(코드) 영역
  - 프로그램 실행 코드를 저장하며 한번 로딩하면 바뀌지 않음
  - 프로그램을 시작할 때 컴파일한 프로그램(기계어)이 저장되어 있음
  - 코드 영역에 저장된 명령어를 하나씩 가져가서 처리함
  - 읽기 전용 영역 → 프로세스가 함부로 변경할 수 없고 변경 시 오류를 발생
- 자료(data) 영역
  - 전역변수, 정적 변수 저장
  - 프로그램이 종료되면 소멸함
  - BSS: 초기화되지 않은 데이터 저장
  - GVAR(Data영역): 초기화된 데이터
    - 초기값이 ROM에 저장된다.
- 스택
  - 자료를 **일시적**으로 저장
  - 함수 매개변수, 반환 주소, 지역 변수 등
  - 컴파일 시간에 크기가 결정되며, 함수 종료 시 메모리 자동 해제
  - 스택 영역에 저장되는 **함수의 호출 정보**를 **스택 프레임**이라고 함
- 힙
  - 프로세스 **수행 시간 동안 동적으로 할당**되는 메모리 영역
  - 스택과 힙 영역은 같은 공간을 공유함 → 힙이 메모리의 낮은 주소부터 할당되면 스택은 높은 주소부터 할당됨
