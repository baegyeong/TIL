## Webpack

- 프로젝트의 구조를 분석하고 자바스크립트 모듈을 비롯한 관련 리소스들을 찾은 다음 이를 브라우저에서 이용할 수 있는 번들로 묶고 패킹하는 모듈 번들러
- 모든 파일을 모듈로 취급하여 하나 또는 여러 개의 번들 파일로 통합

### webpack이 필요한 이유

- 성능 최적화 및 자동화
  - tree shaking을 통한 최적화 수행
  - HTTP 요청 수를 감소하여 웹사이트 성능과 로딩 속도 향상
- 파일 단위의 자바스크립트 모듈 관리
  - 모든 파일으 모듈화하여, 웹 애플리케이션을 구성할 수 있게 해줌
- 번들러가 제공하는 편의성
  - 번들러가 컴파일 과정에서 필요한 플러그인을 추가하고 번들러를 실행

### webpack 동작 방식

- 애플리케이션 전체를 번들링한 후 개발 서버 시작
- 파일 변경 시 관련된 부분만 다시 번들링하는 HMR(Hot Module Replacement)을 지원하지만, 프로젝트 규모가 커질수록 느려지는 경향이 있음

## Vite

- 개발 환경에서 ES Modules를 네이티브로 활용하여 번들링 없이 동작
- esbuild를 사용하여 기존 도구보다 10~100배 빠른 속도
- 프로덕션 빌드에는 Rollup 사용

### Vite 동작 방식

- 개발 서버 시작 시 사전 번들링으로 의존성만 처리하고, 소스코드는 브라우저가 요청할 때 변환하여 제공
- 프로젝트 크기와 무관하게 빠른 초기 구동 속도 유지
